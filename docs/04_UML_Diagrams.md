# UML Диаграммы системы складского учёта

## 1. Диаграмма вариантов использования (Use Case Diagram)

### Актёры системы:
- **Кладовщик** - основной пользователь системы
- **Менеджер склада** - руководитель склада
- **Администратор** - технический специалист
- **Аудитор** - контролирующий сотрудник

### Основные сценарии использования:

#### Для кладовщика:
1. Регистрация прихода товара
2. Оформление расхода товара
3. Перемещение товаров внутри склада
4. Проведение инвентаризации
5. Поиск товаров на складе

#### Для менеджера:
1. Утверждение документов
2. Формирование отчётов
3. Контроль остатков
4. Управление пользователями
5. Анализ работы склада

#### Для администратора:
1. Управление номенклатурой
2. Настройка системы
3. Резервное копирование
4. Мониторинг работы системы

#### Для аудитора:
1. Проверка документов
2. Контроль инвентаризации
3. Формирование актов
4. Анализ расхождений

## 2. Диаграмма последовательности (Sequence Diagram)

### Сценарий: Приёмка товара

```mermaid  
sequenceDiagram
    participant Кладовщик
    participant Система
    participant БД
    participant Склад
    
    Кладовщик->>Система: Создать приходную накладную
    Система->>БД: Проверить существование товара
    БД-->>Система: Данные товара
    Система->>Система: Рассчитать суммы
    Система-->>Кладовщик: Форма накладной
    Кладовщик->>Система: Ввести данные приёмки
    Система->>БД: Сохранить накладную
    Система->>БД: Обновить остатки склада
    БД-->>Система: Подтверждение
    Система-->>Кладовщик: Накладная создана
    Кладовщик->>Склад: Разместить товар
